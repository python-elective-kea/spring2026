# Ændre projektet til at gøre brug af en lokal .env fil
Projektet læser din api key fra `~/.vibe/.env`. Dette er som udgangspunkt godt.    
Vi vil dog gerne have at der i stedet skal læses fra en .env fil i selve repositoriet (for no reason other than for fun).     

**Motivation**: Lav en ny feature til `Mistral-Vibe`

---

* Brug `vibe` til at finde frem til hvordan denne feature fungere nu, hvor i koden det er skrevet, og hvor og hvordan du kan ændre i koden, så du læser fra en lokal .env file i stedet for en global .env fil. 
* Sørg for at sikkerheden er i orden i den kode du implementerer. 
    * Ingen deling af api key på nettet. etc.
* Sørg for at unit tests stadig 'passer'. 
    * Hvorfor er alle test stadig `passed`?
* Sørg for at 'Linting' fungerer
    * Hvad er det der evt. ikke virker mere? 

---

**OBS**: Selvom din kode er skrevet af en LLM, skal du så godt som muligt prøve at forstå hvad der sker. Åben de filer der er blevet ændret kode i og se hvad der er ændret. Brug lidt tid på det!
