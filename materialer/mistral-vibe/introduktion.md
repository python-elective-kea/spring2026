# Mistral og mistral-vibe
I dag arbejder vi på en applikation **forket** og **clonet** fra GitHub.     
Scenariet er meget lig hvad i kunne opleve en af de første dage på jeres nye arbejde.         
Sæt dig ind i den kode vi arbejder på for øjeblikket, lav en ny feature, lav et bugfix, lav et Pull request og få din feature merget ind i master.     

Koden vi kommer til at arbejde med er fra [dette repo](https://github.com/mistralai/mistral-vibe).    
I skal først og fremmest installere og analysere. I skal ændre i koden, og lave et Pull Request til upstream repoet (i hvert tilfælde i teorien).    

## Læringsmål
* Kunne sætte sig ind i hvordan et python projekts github struktur kan være
* Kunne sætte sig ind i koden på et open source project på github.
* Kunne bruge en Agentic CLI til at skabe sig overblik over et projekt.
* Forstå hvornår man bruger `pip`, `uv`, `requirements.txt`, `pyproject.toml` til håndtering af dependencies. 
* Kunne identificere fejl i en kodebase, og fixe dem.
